# 改进计划 4 - CloudKit 降级保障方案

## 项目概述

### 需求背景
当前应用在启动时同步初始化 SwiftData + CloudKit，如果 CloudKit 初始化失败会直接导致应用崩溃（`fatalError`），严重影响用户体验。需要实现完整的降级保障机制，确保应用在任何情况下都能正常启动和使用。

### 目标
- 应用永远不会因为 CloudKit 问题而崩溃
- 使用异步初始化方式，避免阻塞主线程
- 提供降级选项：用户可以选择使用本地存储
- 代码简洁，符合现有风格
- 复用现有代码和组件

---

## 一、逻辑流程设计

### 启动流程
```
1. 应用启动
   ↓
2. 异步初始化 ModelContainer（尝试 CloudKit）
   ├─ ✅ 成功 → 进入主界面
   │           数据同步到 CloudKit
   │
   └─ ❌ 失败 → 显示错误界面
              ├─ [重试] → 重新尝试初始化
              │
              └─ [使用本地存储] → 使用纯本地存储
                                 → 进入主界面
   ↓
3. 主界面正常运行
   ↓
4. 设置中查看 iCloud 同步状态（复用现有 CloudKitStatusView）
```

### 核心原则
- **提供降级选项**：用户可以选择跳过 CloudKit，使用本地存储
- **最小代码原则**：只修改必要的代码，避免过度设计
- **复用现有组件**：使用现有的 CloudKitStatusView 显示同步状态

#### 最终代码实现
```swift
// pethealthdataApp.swift
import SwiftUI
import SwiftData

@main
struct pethealthdataApp: App {
    @State private var modelContainer: ModelContainer?
    @State private var hasError = false
    @State private var errorMessage = ""
    @State private var isInitializing = false
    @AppStorage("useLocalStorage") private var useLocalStorage = false
    
    var body: some Scene {
        WindowGroup {
            Group {
                if let container = modelContainer {
                    MainTabView()
                        .modelContainer(container)
                        .onAppear {
                            requestNotificationPermission()
                            NotificationService.shared.setupNotificationCategories()
                        }
                } else if hasError {
                    ErrorView(
                        message: errorMessage,
                        onRetry: {
                            Task {
                                await initializeModelContainer()
                            }
                        },
                        onUseLocal: {
                            Task {
                                await initializeLocalModelContainer()
                            }
                        }
                    )
                } else {
                    LoadingView()
                        .task {
                            if !isInitializing {
                                await initializeModelContainer()
                            }
                        }
                }
            }
        }
    }
    
    private func initializeModelContainer() async {
        guard !isInitializing else { return }
        isInitializing = true
        
        if useLocalStorage {
            await performLocalInitialization()
            return
        }
        
        do {
            let config = ModelConfiguration()
            let container = try ModelContainer(
                for: Pet.self, VaccineRecord.self, Medication.self, WeightRecord.self, HealthEvent.self,
                configurations: config
            )
            
            await MainActor.run {
                modelContainer = container
                isInitializing = false
            }
            
            print("ModelContainer initialized with CloudKit")
        } catch {
            print("Failed to initialize ModelContainer: \(error)")
            
            await MainActor.run {
                errorMessage = "Unable to connect to iCloud. You can retry or use local storage."
                hasError = true
                isInitializing = false
            }
        }
    }
    
    private func initializeLocalModelContainer() async {
        guard !isInitializing else { return }
        isInitializing = true
        
        await performLocalInitialization()
    }
    
    private func performLocalInitialization() async {
        do {
            let documentsURL = FileManager.default.urls(for: .documentDirectory, in: .userDomainMask).first!
            let localURL = documentsURL.appendingPathComponent("PetHealthData.sqlite")
            
            // 关键：使用 cloudKitDatabase: .none 明确禁用 CloudKit
            let config = ModelConfiguration(url: localURL, allowsSave: true, cloudKitDatabase: .none)
            
            let container = try ModelContainer(
                for: Pet.self, VaccineRecord.self, Medication.self, WeightRecord.self, HealthEvent.self,
                configurations: config
            )
            
            await MainActor.run {
                modelContainer = container
                useLocalStorage = true
                hasError = false
                isInitializing = false
            }
            
            print("ModelContainer initialized with local storage")
        } catch {
            print("Failed to initialize local ModelContainer: \(error)")
            
            await MainActor.run {
                errorMessage = "Failed to initialize storage: \(error.localizedDescription)"
                hasError = true
                isInitializing = false
            }
        }
    }
    
    private func requestNotificationPermission() {
        NotificationService.shared.requestAuthorization { granted in
            if granted {
                print("Notification permission granted")
            } else {
                print("Notification permission denied")
            }
        }
    }
}

// MARK: - Loading View
struct LoadingView: View {
    var body: some View {
        VStack(spacing: 20) {
            Spacer()
            ProgressView()
                .scaleEffect(1.2)
            Text("Loading...")
                .font(.headline)
                .foregroundColor(.secondary)
            Spacer()
        }
        .padding()
    }
}

// MARK: - Error View
struct ErrorView: View {
    let message: String
    let onRetry: () -> Void
    let onUseLocal: () -> Void
    
    var body: some View {
        VStack(spacing: 24) {
            Spacer()
            
            Image(systemName: "icloud.slash")
                .font(.system(size: 50))
                .foregroundColor(.orange)
            
            Text("iCloud Unavailable")
                .font(.title)
                .fontWeight(.bold)
            
            Text(message)
                .font(.subheadline)
                .foregroundColor(.secondary)
                .multilineTextAlignment(.center)
                .padding(.horizontal)
            
            VStack(spacing: 12) {
                Button(action: onRetry) {
                    HStack {
                        Image(systemName: "arrow.clockwise")
                        Text("Retry")
                    }
                    .font(.headline)
                    .foregroundColor(.white)
                    .frame(maxWidth: .infinity)
                    .padding(.vertical, 12)
                    .background(Color.blue)
                    .cornerRadius(12)
                }
                
                Button(action: onUseLocal) {
                    HStack {
                        Image(systemName: "internaldrive")
                        Text("Use Local Storage")
                    }
                    .font(.headline)
                    .foregroundColor(.blue)
                    .frame(maxWidth: .infinity)
                    .padding(.vertical, 12)
                    .background(Color.blue.opacity(0.1))
                    .cornerRadius(12)
                }
            }
            .padding(.horizontal, 32)
            
            Text("Local storage won't sync across devices")
                .font(.caption)
                .foregroundColor(.secondary)
            
            Spacer()
        }
        .padding()
    }
}
```

## 三、技术原理说明

### SwiftData + CloudKit 工作原理

当 entitlements 中启用了 CloudKit 时：
1. `ModelConfiguration()` 默认尝试使用 CloudKit
2. 如果 CloudKit 不可用，可能抛出错误
3. 我们需要提供降级选项，让用户选择使用本地存储

### 本地存储配置（关键）

```swift
// ❌ 错误的配置 - 仍然会尝试 CloudKit
let config = ModelConfiguration(url: localURL, allowsSave: true)

// ✅ 正确的配置 - 明确禁用 CloudKit
let config = ModelConfiguration(url: localURL, allowsSave: true, cloudKitDatabase: .none)
```

### 关键点：
- ✅ **提供降级选项**：用户可以选择跳过 CloudKit
- ✅ **本地存储独立**：使用单独的数据库文件
- ✅ **用户偏好持久化**：使用 @AppStorage 记住用户选择
- ✅ **明确禁用 CloudKit**：使用 `cloudKitDatabase: .none`

## 九、修复经验总结

### 问题 1：状态切换导致 Task 被中断

#### 现象
点击 "Use Local Storage" 按钮后没有任何反应，无法进入主界面。

#### 原因分析
```
用户点击 "Use Local Storage"
    ↓
initializeLocalModelContainer() 开始执行
    ↓
第一行代码: hasError = false
    ↓
视图立即从 ErrorView 切换到 LoadingView
    ↓
LoadingView 的 .task 被触发
    ↓
调用 initializeModelContainer()
    ↓
原来的 initializeLocalModelContainer() Task 被中断 ❌
```

#### 解决方案
添加 `isInitializing` 标志位防止重复初始化：

```swift
@State private var isInitializing = false

private func initializeModelContainer() async {
    guard !isInitializing else { return }
    isInitializing = true
    // ...
}

// LoadingView 的 task 中检查
.task {
    if !isInitializing {
        await initializeModelContainer()
    }
}
```

---

### 问题 2：ModelConfiguration 仍然尝试使用 CloudKit

#### 现象
即使指定了本地 URL，点击 "Use Local Storage" 后仍然无法进入主界面。

#### 原因分析
当 entitlements 中启用了 CloudKit 时，即使指定了本地 URL，SwiftData 仍然会尝试使用 CloudKit：

```swift
// ❌ 这个配置仍然会尝试 CloudKit
let config = ModelConfiguration(url: localURL, allowsSave: true)
```

#### 解决方案
使用 `cloudKitDatabase: .none` 明确禁用 CloudKit：

```swift
// ✅ 正确的配置 - 明确禁用 CloudKit
let config = ModelConfiguration(url: localURL, allowsSave: true, cloudKitDatabase: .none)
```

---

### 问题 3：.task 修饰符不能直接应用于 WindowGroup

#### 现象
编译错误：`value of type 'WindowGroup<...>' has no member 'task'`

#### 原因分析
`.task` 修饰符只能应用于 View，不能直接应用于 Scene（如 WindowGroup）。

#### 解决方案
使用 `Group` 包裹视图，并将 `.task` 应用于内部的 View：

```swift
// ❌ 错误用法
WindowGroup {
    // ...
}
.task {  // 编译错误
    await initializeModelContainer()
}

// ✅ 正确用法
WindowGroup {
    Group {
        // ...
    }
}
// 或者在 LoadingView 上使用 .task
```

---

### 关键经验总结表

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| Task 被中断 | 状态切换导致视图重建 | 添加 `isInitializing` 标志位 |
| 本地存储仍尝试 CloudKit | entitlements 启用了 CloudKit | 使用 `cloudKitDatabase: .none` |
| .task 编译错误 | .task 不能应用于 Scene | 使用 Group 包裹或应用于 View |
